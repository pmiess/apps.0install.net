apps.0install.net - Java

To get the traditional Java commands: 

      $ 0alias java https://apps.0install.net/java/openjdk-jre
      $ 0alias javac https://apps.0install.net/java/openjdk-jdk

The other tools are available by passing the tool name as a third argument, e.g. 

      $ 0alias javadoc https://apps.0install.net/java/openjdk-jdk javadoc

To get common Java build tools: 

      $ 0alias ant https://apps.0install.net/java/ant
      $ 0alias mvn https://apps.0install.net/java/maven
      $ 0alias gradle https://apps.0install.net/java/gradle

When making packages that depend on Java, either add the JRE's bin directory to PATH, or use this pattern: 

      "${MYPROG_JRE_BIN}java" ...

If Zero Install selects a native version of Java (part of your distribution) then MYPROG_JRE_BIN will expand to the empty string and the command will simply invoke java. Otherwise, it will run java from the selected Zero Install implementation.

Available feeds

  • OpenJDK JRE (Runtime)
  • OpenJDK JDK (Developer Kit)
  • Apache Ant (pure Java build tool)
  • Kotlin (statically typed programming language for modern multiplatform applications)
  • Apache Maven (build manager for Java projects)
  • Gradle (modern open source build tool for continuous delivery)
  • Scala (functional and object-oriented programming language based on the Java Virtual Machine)
  • SWT (the Standard Widget Toolkit)
  • HyperSQL (100% Java Database)
  • IRIS Reasoner (Datalog reasoner)
  • SableCC (Parser generator)
